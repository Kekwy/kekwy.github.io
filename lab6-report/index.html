一、实验目的

取得目标靶机的root权限。

我们将使用到以下攻击手段：

  • 主机发现、端口扫描
  • 文件上传漏洞
  • 一句话木马
  • Linux 发行版漏洞本地提权
  • 使用 sqlmap 进行注入
  • 使用蚁剑生成虚拟终端
  • 利用 XMLRPC 客户端注入命令
  • 使用 gdb-peda 调试程序
  • 缓冲区溢出攻击

    继续往下看，发现该函数返回后，所读取的字符串在内存中的首地址为 0xffffdc6c ：

    继续调试，发现调用函数 vuln 时发生缓冲区溢出。

  6. 查看函数 vuln 的汇编代码：

    发现调用了函数 strcpy，将 eax 中地址指向的字符串拷贝到了 ebx 中地址指向的空间。而 ebx 中的值为 ebp - 0x3a，即目标空间的首地址距当前函数栈顶0x3a（58）字节。故当我们输入的字符串的长度大于58时，从第59个字符开始，就会覆盖栈中已有的数据。进行字符串拷贝时，当前函数栈顶之上的部分栈空间结构如下：

    因此，我们输入的第63、64、65、66个字符恰好覆盖了当前函数的返回地址。真正找到了注入点！
